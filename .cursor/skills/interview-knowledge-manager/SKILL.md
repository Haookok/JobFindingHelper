# 面试知识库管理

## 触发场景
- 用户要求添加/编辑面试知识点
- 用户提到八股文、面试题、技术知识点
- 用户要求整理某个技术方向的内容

## 知识库位置
`backend/app/knowledge/` 下按分类存放：

| 目录 | 内容 |
|------|------|
| `java/` | Java 基础、JVM、并发、集合、Spring |
| `python/` | Python 基础、GIL、装饰器、异步、框架 |
| `frontend/` | HTML/CSS/JS、React、Vue、浏览器原理 |
| `algorithms/` | 排序、搜索、DP、图、贪心、分治 |
| `database/` | MySQL、Redis、索引、事务、分库分表 |
| `os/` | 进程线程、内存、IO 模型、死锁 |
| `network/` | TCP/IP、HTTP/HTTPS、DNS、WebSocket |
| `system-design/` | 分布式、微服务、MQ、缓存、限流 |

## 操作流程

### 添加新知识点

1. 确定分类目录
2. 使用以下模板创建 Markdown 文件，文件名用英文短横线命名（如 `jvm-memory-model.md`）

```markdown
# 知识点标题

- **难度**: 基础 | 进阶 | 深入
- **分类**: 具体分类
- **标签**: [标签1, 标签2]

## 核心概念
用 1-3 段话解释清楚原理。

## 详细解析
深入剖析，必要时用图表辅助说明。

## 示例代码
可直接运行的代码，附带注释。

## 面试追问
- 追问 1：考察更深层原理
- 追问 2：考察实际应用
- 追问 3：考察与其他知识的关联

## 常见误区
面试者容易犯错的点。
```

### 编辑已有知识点
1. 读取现有文件内容
2. 保持格式不变，更新具体章节
3. 确保追问部分至少 3 条

## 质量检查
- 概念是否准确无误
- 代码示例是否可运行
- 追问是否覆盖原理/应用/对比三个层面
- 难度标记是否恰当
