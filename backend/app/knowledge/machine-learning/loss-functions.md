# 损失函数详解

- **难度**: 进阶
- **分类**: 机器学习
- **标签**: [损失函数, 交叉熵, 对比学习, 类别不均衡, 面试]

## 核心概念

损失函数就像**考试的评分标准**——它告诉模型"你的答案离正确答案还差多远"。

想象你在练习射箭：每次射完，教练会告诉你偏了多少、偏向哪个方向。这个"偏差反馈"就是损失函数干的事。训练模型，本质上就是不断调整姿势，让这个偏差越来越小。

不同的"考试"需要不同的"评分标准"：预测房价（回归）和判断猫狗（分类）的打分方式完全不同。选对损失函数，模型才知道该往哪个方向努力。

## 详细解析

### 回归任务：MSE、MAE、Huber——三种"量距离"的方式

假设你猜一个数，正确答案是 100：

- **MSE（均方误差）**：算偏差的平方再取平均。你猜 110，扣 100 分；你猜 120，扣 400 分——越偏罚越狠。好处是处处光滑好优化，坏处是一个离谱的答案就能带偏整个成绩。
- **MAE（平均绝对误差）**：直接算偏了多少取平均。猜 110 扣 10 分，猜 120 扣 20 分——线性惩罚，不怕个别离谱值，但在零点处不够"光滑"，优化时可能会抖。
- **Huber**：小偏差时像 MSE（光滑好算），大偏差时像 MAE（不怕极端值），是个折中选手。需要你定一个"分界线" δ。

### 分类任务：交叉熵——"答对给多少分"

你在做选择题，模型给每个选项打了一个"信心分数"。交叉熵的逻辑是：**如果正确答案你给的信心很高，扣分少；信心很低，扣分狠**。

- **二分类**：配合 Sigmoid（把分数压到 0-1），用二元交叉熵（BCE）。
- **多分类**：配合 Softmax（把多个分数变成"概率分布"，加起来等于 1），用多类交叉熵。
- **标签平滑**：相当于告诉模型"别太自信，正确答案给 0.9 的信心就行，留 0.1 分给其他选项"，防止过度自信。

### 类别不均衡：Focal Loss——"重点关注差生"

想象一个班 100 人，95 个学生已经学会了（简单样本），5 个还没学会（难样本）。普通交叉熵会被那 95 个人的反馈淹没，忽略真正需要帮助的 5 个人。

**Focal Loss** 给已经学会的人降低反馈权重，集中精力关注"差生"。参数 γ 控制"差多少才算差生"，α 控制类别本身的权重。目标检测里前景（目标）远少于背景，特别适合用它。

### 度量学习：对比损失与三元组损失——"谁跟谁更像"

不是直接分类，而是学习"相似度"：

- **对比损失**：拿两个样本比——同类的拉近，不同类的推远。就像整理相册，同一个人的照片放一堆，不同人的分开。
- **三元组损失**：一次看三张照片（锚点、同类、不同类），要求锚点离同类近、离不同类远，中间留一个安全距离（margin）。

### KL 散度与 InfoNCE——"两个分布有多不一样"

- **KL 散度**：衡量两个概率分布的差距，就像比较两个班级的成绩分布有多不同。注意它是不对称的：A 看 B 的差距 ≠ B 看 A 的差距。
- **InfoNCE**：对比学习的核心损失，在一批样本中做"配对题"——把正确的一对拉近，其他都推远。SimCLR、MoCo 都用这个。

### 怎么选？一张表搞定

| 场景 | 推荐损失 | 一句话理由 |
|------|----------|------------|
| 回归，数据比较干净 | MSE | 光滑好优化，跟统计理论一致 |
| 回归，有很多异常值 | MAE / Huber | 不怕个别离谱的数据拖后腿 |
| 二分类 | BCE + Sigmoid | 概率输出、梯度好算 |
| 多分类 | CE + Softmax | 多选一标配，数值稳定 |
| 检测 / 严重不均衡 | Focal / 加权 CE | 不让简单样本主导训练 |
| 对比学习 | InfoNCE | 学相似度，拉近正对推远负对 |
| 知识蒸馏 | KL + CE | 同时学老师的"软答案"和真实标签 |

## 示例代码

```python
import numpy as np

# MSE：预测值和真实值差的平方取平均，偏得越多罚得越重
def mse(y_true, y_pred):
    return np.mean((y_true - y_pred) ** 2)

# 二元交叉熵：模型对正确答案越有信心，损失越小
def binary_cross_entropy(y_true, p_pred, eps=1e-7):
    p_pred = np.clip(p_pred, eps, 1 - eps)  # 防止 log(0) 爆炸
    return -np.mean(y_true * np.log(p_pred) + (1 - y_true) * np.log(1 - p_pred))

# Focal Loss：给"已经学会的样本"降权，重点关照难样本
def focal_loss_binary(y_true, p_pred, gamma=2.0, alpha=0.25, eps=1e-7):
    p_pred = np.clip(p_pred, eps, 1 - eps)
    ce = -(y_true * np.log(p_pred) + (1 - y_true) * np.log(1 - p_pred))
    p_t = np.where(y_true == 1, p_pred, 1 - p_pred)
    mod = (1 - p_t) ** gamma       # 预测越准，这个值越小，相当于"降权"
    a_t = np.where(y_true == 1, alpha, 1 - alpha)
    return np.mean(a_t * mod * ce)

# Softmax：把一组原始分数变成概率分布（加起来等于 1）
def softmax(logits):
    z = logits - np.max(logits, axis=-1, keepdims=True)  # 减最大值防溢出
    e = np.exp(z)
    return e / np.sum(e, axis=-1, keepdims=True)

# 多类交叉熵：看模型给正确类别分配了多少概率
def cross_entropy_multiclass(y_onehot, logits):
    p = softmax(logits)
    return -np.mean(np.sum(y_onehot * np.log(p + 1e-7), axis=-1))

# Huber：小误差像 MSE 一样光滑，大误差像 MAE 一样淡定
def huber_loss(y_true, y_pred, delta=1.0):
    err = y_true - y_pred
    abs_err = np.abs(err)
    quad = 0.5 * err**2
    linear = delta * (abs_err - 0.5 * delta)
    return np.mean(np.where(abs_err <= delta, quad, linear))
```

## 面试追问

- 面试官可能会这样问你：Softmax + 交叉熵对 logits 的梯度形式特别简洁（就是 p - y），你能从推导角度解释为什么吗？
- 面试官可能会这样问你：Focal Loss 里 γ 和 α 分别控制什么？如果 γ=0 会退化成什么？
- 面试官可能会这样问你：对比损失和三元组损失在"挖掘难样本"这件事上有什么区别？
- 面试官可能会这样问你：KL(P||Q) 和 KL(Q||P) 不一样大，在变分推断里一般用哪个方向？为什么？
- 面试官可能会这样问你：如果回归任务的标签分布很长尾或者噪声很多，你会怎么改损失函数？

## 常见误区

- 很多人会搞混的地方：**二元交叉熵（BCE）和多类交叉熵（CE）** 不是一回事。二分类用单个输出 + Sigmoid + BCE，多分类用多个输出 + Softmax + CE，接口和标签格式完全不同，混用必出 bug。
- 很多人会搞混的地方：以为 **Focal Loss 是万能的**。极端不均衡时，光靠它不够，还得配合采样策略、类别权重或两阶段检测。
- 很多人会搞混的地方：手写 `log(softmax(x))` 容易数值爆炸。实际中要用 log-softmax 或框架内置实现，利用 log-sum-exp 技巧保数值稳定。
- 很多人会搞混的地方：**多标签**（一个样本可以同时属于多个类别）误用 Softmax+CE。应该对每个标签独立做 Sigmoid+BCE，因为 Softmax 有"互斥"假设。
- 很多人会搞混的地方：对比学习只背 InfoNCE 公式，却不理解温度系数 τ。τ 太小梯度太尖锐，τ 太大区分度太弱。
