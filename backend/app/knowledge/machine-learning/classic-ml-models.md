# 经典机器学习模型

- **难度**: 基础
- **分类**: 机器学习
- **标签**: [线性模型, SVM, 树模型, 聚类, PCA, 偏差方差, 面试]

## 核心概念

经典机器学习模型就像一套**分类工具箱**——每种工具有自己擅长的活儿：

- **线性模型**像一把直尺，画一条线把东西分开，简单直观；
- **SVM** 像一把精确的激光切割机，找到最宽的那条分界线；
- **决策树**像一个问答筛子，一层层地问"这个特征大于多少？"；
- **随机森林**像一群人投票，少数服从多数，比一个人判断靠谱；
- **XGBoost/LightGBM** 像接力赛，每个选手专门弥补上一个人的短板。

没有万能模型，关键是看你的数据长什么样、需要什么。

## 详细解析

### 线性回归与逻辑回归——直尺画线

**线性回归**：用一条直线（高维是平面）去拟合数据，目标是让预测值和真实值的偏差最小。可以加惩罚防过拟合：L1（Lasso）会让一些权重变成 0 实现"自动选特征"，L2（Ridge）让权重都小一点保持稳定。

**逻辑回归**：名字里有"回归"但干的是分类的活儿。它在线性打分后加一个 Sigmoid 函数，把分数压到 0-1 之间变成概率。"你得癌症的概率是 0.8"——这种需要概率输出的场景特别适合。

### SVM——激光切割机

SVM 的核心思想：在两类数据之间找一条分界线，而且要让这条线离两边最近的点**尽可能远**（最大化间隔）。就像你在两群人之间画一条路，路越宽越安全。

数据在低维空间分不开怎么办？**核函数**帮你把数据"抛"到高维空间，在那里可能就能分开了。常用 RBF（高斯核），但参数 γ 要调好，太大容易过拟合。

现在大数据时代，SVM 用得少了，但面试爱考它的理论（对偶、支持向量、核技巧）。

### 决策树——问答筛子

一棵决策树就是一连串的 if-else 问题："年龄 > 30？""收入 > 5万？"每个问题把数据分成两拨，直到叶子节点给出最终答案。

怎么决定先问哪个问题？用**信息增益**或 **Gini 系数**——哪个问题能把数据分得最"纯"，就先问。单棵树容易过拟合（记住了训练数据的噪声），需要剪枝或限制深度。

### 随机森林——投票团

一棵树容易犯错？那就种一片森林！随机森林训练很多棵树，每棵树只看部分数据和部分特征，最后投票决定。这就是 **Bagging** 的思想——通过多人投票来降低"方差"（不稳定性）。

### XGBoost / LightGBM——接力赛选手

和随机森林不同，**梯度提升树（GBDT）** 是串行的：第一棵树先做预测，第二棵树专门修正第一棵的错误，第三棵修正第二棵的……像接力赛一样，每个选手弥补前一个人的不足。这叫 **Boosting**——通过持续纠错来降低"偏差"（系统性偏离）。

- **XGBoost**：引入二阶泰勒展开和正则化，生态成熟，竞赛老牌冠军。
- **LightGBM**：用 leaf-wise 生长策略和直方图近似，速度更快，大数据场景首选。

**在表格数据上，GBDT 系至今仍常胜于深度学习。**

### K-Means 与 DBSCAN——自动分组

- **K-Means**：你告诉它分几组（K），它反复调整每组的中心点，直到稳定。适合球形、大小均匀的簇，对离群点敏感。选 K 可以看"肘部法"或轮廓系数。
- **DBSCAN**：不用指定 K，自动根据密度分组，还能发现形状奇怪的簇和识别噪声点。但参数 ε 和 MinPts 需要调好。

### PCA 降维——信息压缩

数据有 100 个特征太多了？PCA 帮你找到最重要的几个"方向"（主成分），把 100 维压到 10 维，同时尽量保留信息。就像把一篇长文章概括成几个关键句。它是线性的；非线性降维可以看 t-SNE（适合可视化，但不保距离关系）。

### 偏差-方差权衡——模型选择的核心矛盾

- **偏差高**（欠拟合）：模型太简单，学不到规律，就像用直线拟合一条弯曲的数据。
- **方差高**（过拟合）：模型太复杂，把噪声也记住了，换个数据集就不灵了。

增加复杂度能降偏差但升方差，反过来也是。交叉验证、正则化、集成方法都是在这两者之间找平衡。

## 示例代码

```python
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.cluster import KMeans, AgglomerativeClustering
from sklearn.decomposition import PCA
from sklearn.model_selection import cross_val_score

# 逻辑回归 vs 随机森林，5 折交叉验证比一比谁更准
def quick_eval(X, y):
    lr = LogisticRegression(max_iter=1000)
    rf = RandomForestClassifier(n_estimators=100, random_state=42)
    s_lr = cross_val_score(lr, X, y, cv=5).mean()
    s_rf = cross_val_score(rf, X, y, cv=5).mean()
    return {"logreg_cv": s_lr, "rf_cv": s_rf}

# PCA：保留 95% 的信息，把高维数据"压扁"
def pca_95(X):
    pca = PCA(n_components=0.95)
    return pca.fit_transform(X), pca

# 层次聚类：不像 K-Means 只能分球形簇，它自底向上合并最近的组
def hierarchical_labels(X, n_clusters):
    model = AgglomerativeClustering(n_clusters=n_clusters, linkage="ward")
    return model.fit_predict(X)
```

## 面试追问

- 面试官可能会这样问你：SVM 为什么只有少数"支持向量"就能决定分界线？从对偶和 KKT 条件角度解释一下。
- 面试官可能会这样问你：Bagging（随机森林）和 Boosting（XGBoost）分别在偏差和方差上起什么作用？
- 面试官可能会这样问你：XGBoost 用二阶泰勒展开有什么好处？比只用一阶的 GBDT 强在哪？
- 面试官可能会这样问你：K-Means 是"硬分配"（一个点只属于一个簇），GMM 是"软分配"（概率属于多个簇），这两者怎么对比？
- 面试官可能会这样问你：PCA 的第一主成分是什么意思？它跟原特征空间的旋转有什么关系？

## 常见误区

- 很多人会搞混的地方：**逻辑回归是分类模型不是回归模型**！名字带"回归"是因为它用了回归的思路（线性打分），但输出是概率，目标是分类。
- 很多人会搞混的地方：随机森林的**"特征重要性"**对高基数类别特征和相关特征有偏，不能盲信，解释时要谨慎。
- 很多人会搞混的地方：以为 **DBSCAN 能自动找到最优簇数**。它只是不需要指定 K，但 ε 和 MinPts 调不好照样全盘皆输。
- 很多人会搞混的地方：把 **t-SNE 的二维图上距离当真**。t-SNE 是为了可视化，点之间的距离不能拿来做聚类或其他下游任务。
- 很多人会搞混的地方：比较 XGBoost 和 LightGBM 只会说"LightGBM 更快"。面试要能说出 leaf-wise vs level-wise、直方图近似等具体差异。
- 很多人会搞混的地方：用带正则的线性模型或 K-Means/SVM 前**不做特征标准化**，量纲不同会严重影响结果。
