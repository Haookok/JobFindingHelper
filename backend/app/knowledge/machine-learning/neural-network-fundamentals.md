## 核心概念

神经网络就像一条**层层加工的流水线**。

想象一个工厂：原材料（输入数据）进来，经过一道道工序（网络层），每道工序都做一点加工（线性变换 + 非线性激活），最终出成品（预测结果）。如果成品质量不好（损失大），工厂会从最后一道工序开始，一道道往回查"是哪个环节出了问题"——这就是**反向传播**。

这条流水线能不能跑好，取决于几个关键要素：激活函数让每道工序能做"非线性"加工；归一化让流水线上的物料不会忽大忽小；残差连接像一条"快速通道"，让信息可以跳过某些工序直达后面。

## 详细解析

### 反向传播——从后往前"追责"

前向传播很好理解：数据从左到右过每一层，最后算出损失。反向传播是反过来：从损失出发，用链式法则一层层往回算"每个权重对损失贡献了多少"。

打个比方：期末考试考砸了（损失大），你复盘时会想——是最后的大题没写好？还是中间的计算出错了？还是一开始的概念就没理解？反向传播就是这个"从后往前逐步追责"的过程。

框架（PyTorch 等）帮你自动构建计算图，每个操作只需定义"怎么算"和"怎么求导"，反向传播自动完成。

### 激活函数——流水线上的"变形金刚"

如果每道工序只做线性变换（乘权重加偏置），那无论堆多少层，整体还是线性的——就像一个只会等比例放大缩小的复印机。激活函数引入非线性，让网络能学到弯弯曲曲的复杂关系。

- **ReLU**：负数变 0，正数不变。简单高效，就像一个"只让正数通过"的阀门。缺点是负数永远过不来（死神经元）。
- **Leaky ReLU**：负数不完全变 0，而是乘一个很小的系数（比如 0.01），给负数留条活路。
- **GELU**：更平滑的"门控"，Transformer（BERT 等）的标配。可以理解为"根据输入值的大小概率性地决定让不让通过"。
- **SiLU/Swish**：输入乘以自己的 Sigmoid，自带门控效果，有时比 ReLU 效果更好。

### 归一化——让流水线物料保持一致

没有归一化，经过几层加工后数据的大小可能变得天差地别（有的值 1000，有的 0.001），导致训练不稳定。

- **BatchNorm**：在一个 batch 内，统计同一个特征的均值和方差，然后标准化。就像工厂每批次物料都重新校准一下。问题是 batch 太小统计不准，而且推理时不能用当前 batch 统计（因为可能只有一个样本）。
- **LayerNorm**：对每个样本自己的所有特征做标准化，跟 batch 大小无关。Transformer 用它，因为序列长度不固定，BatchNorm 不方便。
- **RMSNorm**：LayerNorm 的简化版，只除以均方根，不减均值，计算更快。一些大语言模型（如 LLaMA）用它来加速。

三者都有可学习的缩放参数，让网络在标准化的基础上还能调整表达。

### Dropout——随机关掉一些工人

训练时随机让一部分神经元"休息"（输出置零），迫使网络不能过度依赖某几个神经元。就像考试时老师随机抽掉几个学霸，逼其他同学也得学会。推理时所有人都上班，但产出要乘以 (1-p) 做补偿（或者训练时用 inverted dropout 提前除好）。

**关键坑：推理时一定要调用 `model.eval()` 关掉 Dropout，否则指标会崩。**

### 权重初始化——流水线的起点很重要

权重全初始化为 0？所有神经元学到的东西一模一样（对称性），相当于只有一个神经元在干活。

- **Xavier/Glorot 初始化**：适配 tanh/Sigmoid，让每层输出的方差大致稳定。
- **Kaiming/He 初始化**：专门为 ReLU 设计，考虑了负半轴被"砍掉"导致方差减半的问题。深层 CNN 必备。

### 梯度消失与爆炸——信号在流水线上衰减或失控

反向传播时梯度是逐层相乘的。如果每层的乘法因子持续小于 1，梯度就会指数级衰减（消失）；持续大于 1 就会指数级增长（爆炸）。

消失意味着前面的层"收不到反馈"学不动；爆炸意味着参数一步跳到天上去。解决办法：ReLU（导数是 1 或 0，不缩小）、残差连接、归一化、好的初始化、梯度裁剪（针对爆炸）。

### 残差连接——信息高速公路

\(y = F(x) + x\)：输出 = 加工后的结果 + 原始输入。即使加工环节 F(x) 学到的东西接近 0，信息也能通过 "+x" 这条捷径直达后层。

就像工厂流水线旁边有一条传送带，原材料可以直接跳到后面的工序，不用每道工序都经过。这让几百层的网络也能训练，是 ResNet 和所有现代深层网络的基石。

### Pre-LN vs Post-LN——归一化放哪儿

- **Post-LN**（原始 Transformer）：先做子层计算，再做 LayerNorm。深层时训练有时不太稳。
- **Pre-LN**（现代主流）：先做 LayerNorm，再进子层。梯度路径更直接，训练更稳定。

面试能画出一个 Transformer Block 里 Add 和 Norm 的先后位置就够了。

### 激活函数速查表

| 函数 | 一句话特点 | 常用场景 |
|------|------------|----------|
| ReLU | 简单快速，负数直接砍 | CNN 默认选择 |
| Leaky ReLU | 给负数留条活路 | 缓解死神经元 |
| GELU | 平滑的概率门控 | Transformer 标配 |
| SiLU/Swish | 自带门控的平滑曲线 | 部分 ConvNet |

## 示例代码

```python
import numpy as np

# ReLU：负数变 0，正数不变——最简单的非线性"阀门"
def relu(x):
    return np.maximum(0, x)

def relu_grad(x):
    return (x > 0).astype(np.float64)  # 正数导数 1，负数导数 0

# GELU 近似实现：Transformer 里常用的平滑激活函数
def gelu_approx(x):
    return 0.5 * x * (1 + np.tanh(np.sqrt(2 / np.pi) * (x + 0.044715 * x**3)))

# LayerNorm：对每个样本自己的特征做标准化，跟 batch 大小无关
def layer_norm(x, gamma, beta, eps=1e-5):
    mean = x.mean(axis=-1, keepdims=True)
    var = x.var(axis=-1, keepdims=True)
    x_hat = (x - mean) / np.sqrt(var + eps)
    return gamma * x_hat + beta

# Xavier 初始化：让每层输出的方差保持稳定，适配 tanh/Sigmoid
def xavier_uniform(fan_in, fan_out):
    limit = np.sqrt(6 / (fan_in + fan_out))
    return np.random.uniform(-limit, limit, size=(fan_in, fan_out))

# Kaiming 初始化：专为 ReLU 设计，考虑了负半轴被砍的方差损失
def kaiming_uniform(fan_in, fan_out):
    limit = np.sqrt(6 / fan_in)
    return np.random.uniform(-limit, limit, size=(fan_in, fan_out))

# Dropout：训练时随机关掉一些神经元，推理时记得关掉！
def dropout_mask(x, p, train=True, rng=None):
    if not train or p <= 0:
        return x
    rng = rng or np.random.default_rng()
    keep = rng.binomial(1, 1 - p, size=x.shape) / (1 - p)  # inverted dropout
    return x * keep
```

## 面试追问

- 面试官可能会这样问你：推导一层"线性 + Sigmoid"对权重的梯度，并解释为什么 Sigmoid 饱和区会导致梯度消失。
- 面试官可能会这样问你：BatchNorm 训练和推理时的行为有什么不同？为什么推理不能用当前 batch 的统计量？
- 面试官可能会这样问你：如果把残差块的 F(x) 缩小到 0.1*F(x)，或者去掉 BN，训练会怎样？
- 面试官可能会这样问你：Transformer 里为什么用 LayerNorm 而不是 BatchNorm？
- 面试官可能会这样问你：Dropout 和 L2 正则、数据增强都是"正则化"，但它们的机制有什么不同？

## 常见误区

- 很多人会搞混的地方：以为 **BatchNorm 总能加速训练**。极小 batch 或分布式训练时，每个设备上的 BN 统计量噪声太大，效果反而变差。
- 很多人会搞混的地方：**GELU 和 SiLU 混为一谈**。它们长得像但定义不同，面试时要说清楚各自的公式和常用场景。
- 很多人会搞混的地方：以为**梯度裁剪能解决梯度消失**。裁剪只防爆炸；消失要靠 ReLU、残差、Norm、好的初始化。
- 很多人会搞混的地方：**权重全初始化为零**。所有神经元会学到一模一样的东西（对称性无法打破），几乎总是错误的做法。
- 很多人会搞混的地方：**推理时忘了调用 `model.eval()`**，导致 Dropout 还在随机丢神经元、BN 还在用当前 batch 统计量，指标直接崩盘。
